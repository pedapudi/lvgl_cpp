Objective: Detect slow memory leaks or fragmentation over repeated create/destroy cycles.
Methodology: Runs 100 cycles. In each cycle, it creates a lvgl::Screen, populates it with 20 lvgl::Button objects (each with a lambda callback), processes events via lv_timer_handler, and then destroys the screen. This tests the destructors and parent-child cleanup mechanisms.
Scale: 2,000 total allocations/deallocations.

fragmentation

Objective: Stress the heap allocator with randomized allocation patterns to measure fragmentation overhead from C++ closures.
Methodology: Runs 50 iterations with an inner loop of 50 operations. Operations are randomized:
  • Alloc (33%): Creates a button and attaches a lambda capturing a 64-byte struct CaptureState.
  • Free (33%): Deletes a random existing button (and its closure).
  • No-op (33%): Idle.
Comparison: Compares against C code creating buttons with standard C callbacks (no capture context), highlighting the specific cost of stateful C++ closures.
